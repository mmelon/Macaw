Correct approach to Macaw http://forum.macaw.co/discussion/comment/6231/#Comment_6231 Quick analyse of a file ( http://forum.macaw.co/discussion/comment/6874/#Comment_6874 ) If you have to build a whole website using only Macaw… (even if it's not it's main purpose) 1. draw something first (on paper) it will give you a structure a list of element you need to realize, 2. realize a quick wireframe inside Macaw just to know how many element you have to design and how they will react together (with your breakpoints). No images, no details only the essential: the structure. The goal here is to determine what sizes your element should have and how they will react together. 3. prepare all your differents assets (img, svg, font), name them and optimise them, 4. create a new file (with your wireframe in mind or open in a browser), 5. draw the structure of your final HTML (header, section, articles, footer, etc.) 6. draw each one of your element in separate pages (no repetition means better CSS), the wireframe should have gave you all the info you need, 7. if you have some embed CSS you should integrate them inside your main CSS file, 8. take your publishing files and merge them for building your final website, 9. repeat, modify manually if you have to (yeah, in a text editor), 10. That's it! You've build a webpage with Macaw! You should normally make some minor adjustments but you can live without them. Anyway, it's a good thing to test your website in some great online tools like https://developers.google.com/speed/pagespeed/insights/ or http://validator.w3.org/ They will give you lots of precious informations for free! 11. Optional - You can post your creation here Important notes: - the greatest thing in Macaw is the “consolidate page styles”. Use it intelligently and you will have a great CSS file… - … but for that you must name all your elements (read “How to correctly name my elements” below), - don't repeat anything in your page, that's not the purpose of HTML anyway - stay simple, use static positioning only, absolute is a last case scenario, - build your page bit by bit (be patient and rigorous) you'll gain lots of your time at the end, - test you page at each step, the dev tools of your browser is your best friend and it will tell you your mistakes and where the adjustments should be made. How to correctly name my elements? http://forum.macaw.co/discussion/comment/6231/#Comment_6267 How to import my own external CSS file? Link your external CSS file in your header (help). … How to import my own external Javascript file? Link your external javascript file in your header (help). … How to import components or objects from another file? Copy the element you want in your open project, then open your target file and paste it. Switching between projects is not the best way to reuse your elements but it’ll do the job. Note: depending on your breakpoints, an alert window will ask you if you want to create breakpoint if the origin of the file has a larger one than the actual. You can bypass this alert by using the “Edit > Paste Simple" command, or SHIFT + CMD + V. Note: the paste function works only once.
