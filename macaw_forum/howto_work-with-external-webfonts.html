How to work with external webfonts. Macaw uses the PostScript name of your font which some time differ from its full name or file name. Plus in your case the fontsquirrel font generator lowered cased its name… . Example with the excellent M+ WEB FONTS (http://mplus-fonts.sourceforge.jp) Download it here (select the “Webfont Kit” tab > “Download @Font-Face Kit”) You can see that Macaw use the correct font family name inside the font list. Now, when you publish your code: body { background-color: rgb(255, 255, 255); font: normal 1em/1.38 mplus-1c-light; color: rgb(0, 0, 0); } You can see that Macaw now uses the PostScript name. This is the name you need to use in your Head & Tail section as your font-family name. In my Head & Tail section I can now link my file like this: @font-face { font-family: 'mplus-1c-light'; … } So, next time, all you have to do is publish your code and see what Macaw expect for the font family name. Example: Museo_Sans500 in font-family: 'Museo_Sans500'; is the name used by Macaw not the name of your font or font file. When you look at your CSS, it clearly use the same font name (Museo_Sans500) ? Something like font: normal 1em/1.38 Museo_Sans500;. And your directory must be something like this: /your project folder/ |--yourfile.mcw |--yourfile | |--css | | |--index.css | | |--standardize.css | |--index.html | |--js | | |--jquery-min.js |--fonts |--MuseoSans_500.eot |--museosans_500-webfont.woff2 |--MuseoSans_500.woff |--MuseoSans_500.ttf |--MuseoSans_500.svg
